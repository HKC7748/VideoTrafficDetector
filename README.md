## 项目简介

本系统基于 **YOLOv8** 目标检测模型和 **PySide6** 图形界面框架，实现对交通监控视频中的车辆进行实时检测、跟踪、统计与分析。支持自定义检测区域（车道）、车辆类型参数、透视变换校正，并能够导出详细的交通流数据（流量、占有率、排队长度、车头时距等）以及车辆轨迹。

适用于智能交通、路况监测、学术研究等场景。

## 功能特性

- **可视化配置界面** – 图形化编辑检测计划（YOLO模型路径、视频源、车道定义、透视/平行点、车辆尺寸等）
- **实时检测与跟踪** – 利用 ByteTrack 多目标跟踪算法，逐帧检测车辆并保持 ID 一致性
- **车辆属性识别** – 自动识别车辆类型（car、bus、truck 等）及车身颜色（红、黄、蓝、白、黑、灰等）
- **透视校正** – 通过透视变换将图像坐标映射到俯视平面，消除镜头畸变影响
- **车道级统计分析** – 每个车道独立计算：
  - 时间占有率、空间占有率
  - 交通流量（辆/分钟）
  - 排队长度
  - 断面过车数据（速度、车头时距、间距）
- **轨迹拟合与平滑** – 采用卡尔曼滤波 + 线性插值生成连续轨迹，剔除抖动噪声
- **性能评估** – 自动计算实际采样帧率、检测成功率、处理耗时等指标
- **丰富的可视化** – 饼图（车型/颜色/车道分布）、柱状图、数据表格、帧回放（检测框 + 拟合轨迹）
- **一键导出** – 将所有统计数据（CSV 格式）导出到指定文件夹，方便后续分析

## 环境要求

- Python 3.11 及以上
- 操作系统：Windows / macOS / Linux
- 硬件：建议配备 NVIDIA GPU（CUDA 加速），CPU 也可运行但速度较慢

## 安装步骤

1. **克隆或下载本项目**  
   ```bash
   git clone https://github.com/HKC7748/VehicleDetectionTrafficAnalysis.git
   ```

2. **创建虚拟环境（推荐）**  
   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # macOS/Linux
   source venv/bin/activate
   ```

3. **安装依赖**  
   ```bash
   pip install -r requirements.txt
   ```

4. **准备模型文件**  
   下载 YOLOv8 预训练模型（如 `yolov8n.pt` 或 `yolov8s.pt`），放入项目 `./Model/` 目录。  
   也可使用自己微调的模型。

5. **准备资源文件**  
   - 将待分析的视频文件放入 `./Resource/` 目录（支持 .mp4, .avi, .mov）。  
   - （可选）将地图底图 `map.png` 放入 `./Resource/`，用于轨迹可视化背景（若不提供，程序会生成灰色画布）。

## 使用方法

### 第一步：运行程序

```bash
python main.py
```

### 第二步：配置检测参数

在打开的窗口中，切换到 **配置** 标签页：

1. **导入配置**（可选）：点击“导入配置”，选择已有的 JSON 配置文件快速加载参数。
2. **选择模型**：点击 YOLO 模型路径旁的“浏览”，选择 `.pt` 模型文件。
3. **选择视频**：点击视频源旁的“浏览”，选择待分析的视频文件。
4. **设置车辆类型**：在“车辆类型”表格中添加或修改车辆尺寸（长度、宽度），用于空间占有率计算。
5. **设置透视/平行点**：在“透视/平行点”区域添加对应的坐标点（至少4对），用于透视变换校正。  
   - 透视点：图像坐标系下的点。  
   - 平行点：对应俯视图下的点。
6. **添加车道**：点击“添加”按钮创建车道，填写车道名称、检测线点（2×2）和边界点（N×2）。  
   - 检测线：车辆经过此线时记为一次过车事件。  
   - 边界点：定义车道区域，用于占有率计算。
7. **调整统计窗口**：可根据需要修改时间占有率窗口、空间占有率窗口、流量统计窗口、排队长度窗口（单位：秒）。

> **提示**：配置完成后可点击“导出配置”保存为 JSON 文件，下次直接导入复用。

### 第三步：开始检测

切换到 **检测** 标签页：

1. 点击“开始检测”按钮，系统将自动逐帧处理视频。
2. 观察实时输出日志，可看到每帧检测到的车辆数量及处理耗时。
3. 如需提前终止，点击“取消检测”。
4. 检测完成后，右侧输出区会自动显示统计摘要（总车辆数、提取成功率、速度、帧率评分等）。

### 第四步：查看结果

切换到 **可视化** 标签页：

- **控制台**：显示车型、颜色、车道分布的饼图。
- **拟合轨迹**：拖动帧选择滑块，查看不同时刻的检测画面（带检测框）和拟合轨迹图（俯视平面）。
- **各车道数据**：每个车道独立标签页，内含时间占有率、空间占有率、流量、排队长度、断面过车数据的柱状图与表格。
- **一键导出**：点击“一键导出所有数据”，选择保存目录，所有统计数据将按类别导出为 CSV 文件（编码 UTF-8-SIG）。

## 自定义配置说明

### 车辆类型参数

在配置界面的“车辆类型”表格中，可添加任意类型车辆及其长宽（单位：米）。系统会根据这些参数计算空间占有率和车辆投影面积。

### 透视变换点

透视点与平行点必须一一对应，且至少需要 4 对点才能解算单应矩阵。建议选取道路上的明显标记（如车道线端点、标志牌等），平行点的坐标单位应与实际距离一致。

### 检测间隔

目前代码中固定为每 3 帧检测一次（`detectInterval = 3`），可在 `VehicleDetectionSystem.py` 的 `StartDetection` 方法中修改。

### 颜色识别阈值

颜色分类基于 HSV 空间的阈值，位于 `Tool.py` 的 `ClassifyRgbCenter` 函数中，可根据实际场景微调。

## 常见问题

### Q1: 检测时提示“找不到模型文件”

请确保模型文件放置在 `./Model/` 目录下，并在配置界面正确选择了路径。

### Q2: 检测速度慢

使用 GPU 版本的 PyTorch/CUDA 加速。 
使用更轻量的模型（如 `yolov8n.pt`）。

### Q3: 透视变换后轨迹严重变形

检查透视点和平行点是否匹配准确，建议至少选取 6 对均匀分布的点以提高精度。

### Q4: 导出 CSV 乱码

CSV 文件使用 UTF-8-SIG 编码，Excel 打开时请选择“数据→从文本/CSV 导入”，并指定编码为 UTF-8。
